## 3. ОНБОРДИНГ ЗАКАЗЧИКА — ЧЕКЛИСТ

### БЛОК 1: О КОМПАНИИ (5 мин)
1. Как называется компания?
2. Чем занимаетесь? (1-2 предложения)
3. Кто ваш типичный клиент?
4. Какой средний чек?
5. Сколько клиентов в день/неделю?

### БЛОК 2: ПРОДАЖИ (10 мин)
6. Как сейчас продаёте? (WhatsApp, звонки, офлайн)
7. Кто отвечает клиентам сейчас?
8. Какие вопросы задают чаще всего? (ТОП-5)
9. Какие возражения слышите? ("дорого", "подумаю")
10. Как обычно отвечаете на эти возражения?
11. Есть ли готовые скрипты/шаблоны?

### БЛОК 3: ПРОЦЕССЫ (5 мин)
12. Как записываете клиентов? (CRM, тетрадка, WhatsApp)
13. Как принимаете оплату? (Kaspi, карта, наличные)
14. Кто принимает решения по скидкам?
15. Куда слать эскалации? (WhatsApp, Telegram)

### БЛОК 4: КОНТЕНТ (10 мин)
16. Есть ли прайс-лист?
17. Есть ли описание услуг/товаров?
18. Есть ли отзывы/кейсы клиентов?
19. Есть ли фото/видео?
20. Что бот НЕ должен говорить? (запрещённые темы)

### БЛОК 5: ОЖИДАНИЯ (5 мин)
21. Что для вас успех через месяц?
22. Что бот точно должен уметь?
23. Что бот точно НЕ должен делать?
24. Готовы ли отвечать на эскалации в течение 10 минут?

## 4. ЭТАЛОН ДОКУМЕНТА

```markdown
# [ТЕМА]: Возражение "Дорого"

## СИТУАЦИЯ
Клиент говорит что цена высокая, но ещё не отказался.

## ЦЕЛЬ
Показать ценность, не давать скидку.

## ЗАПРЕЩЕНО
- Давать скидку (только менеджер)
- Спорить с клиентом
- Говорить "у всех так"

## СКРИПТ
1. Уточни: "Дорого по сравнению с чем?"
2. Покажи ROI: "Менеджер стоит X, бот — Y. Экономия Z"
3. Предложи тест: "7 дней бесплатно — увидите разницу"

## ПРИМЕР ДИАЛОГА
К: "50 тысяч в месяц это много"
Б: "Понимаю. Сколько стоит менеджер который отвечает ночью? Бот работает 24/7 без отпуска. Давайте 7 дней бесплатно — сами оцените."

## ЕСЛИ НЕ СРАБОТАЛО
После 2 попыток: "Понял вас. Если передумаете — напишите."
НЕ ДАВИТЬ.
```

## 5. ЦИКЛ ОБУЧЕНИЯ

```
1. Бот не знает → эскалация менеджеру
2. Менеджер отвечает
3. Ответ сохраняется в БД
4. Модерация (да/нет/исправить)
5. Если ДА → в базу знаний
6. Бот теперь знает

МЕТРИКА: % вопросов без эскалации
Цель: 70% → 85% → 95%
```

## 8. КРИТИЧЕСКАЯ ВАЛИДАЦИЯ

### ФАКТЫ (подтверждённые данные):

| Проблема | Источник | Данные |
|----------|----------|--------|
| 75% ботов не справляются со сложными запросами | Chanl.ai | Исследование |
| 27% галлюцинаций + 46% фактических ошибок | Medium | Исследование |
| Только 12-15% ботов достигают высокой конверсии | Workhub.ai | Статистика |
| 54% клиентов считают ботов неэффективными | Forrester | Исследование |

### РЕАЛЬНЫЕ ПРОВАЛЫ:

| Компания | Что случилось |
|----------|---------------|
| Air Canada | Бот дал неверную информацию → суд |
| DPD | Бот материться → выключили |
| Chevy | Бот предложил машину за $1 |
| Dashly | $35k потеряно → бот мешал продажам |

### МАРКЕТИНГОВЫЙ ПИЗДЁЖ:

| Заявление | Реальность |
|-----------|------------|
| "Конверсия +67%" | Только ТОП-20%, средняя +23% |
| "ROI 8x" | "Некоторые", не средний |
| "+40% конверсия" | "up to" — максимум |

### РЕАЛЬНЫЕ ЦИФРЫ:

| Метрика | Данные |
|---------|--------|
| Средняя конверсия e-commerce | 1.65-2% |
| С хорошим ботом | 10%+ |
| Средний рост с ботом | +23% |

### КРИТИЧЕСКИЕ ОГРАНИЧЕНИЯ:

Бот НЕ МОЖЕТ:
1. Понять контекст между сессиями
2. Рассуждать логически
3. Чувствовать эмоции клиента
4. Обрабатывать нестандартные ситуации

### ЗАЩИТНЫЕ МЕРЫ:

| Риск | Защита |
|------|--------|
| Галлюцинация | RAG only, не генерация из головы |
| Застревание | Макс 3 попытки → эскалация |
| Обещания | Whitelist, нет импровизации по деньгам |
| Агрессия | Sentiment → сразу человеку |

## 9. ГЛУБОКИЙ АНАЛИЗ (критическая валидация #2)

### ЧТО ОТЛИЧАЕТ УСПЕШНЫХ ОТ ПРОВАЛЬНЫХ:

| Успешные | Провальные |
|----------|------------|
| Специализация (1 бот = 1 задача) | Один бот на всё |
| Метрики = продажи | Deflection rate |
| Прозрачность (говорит когда не знает) | Уверенно врёт |
| Эскалация работает | Клиент застревает |
| Продуктовое знание | "Обратитесь к менеджеру" |

**Источник:** HBR Case Study, Alibaba (5 специализированных ботов)

### ГАЛЛЮЦИНАЦИИ — КАК ПРЕДОТВРАТИТЬ:

| Метод | Эффективность |
|-------|---------------|
| RAG only | До 20% ошибок |
| RAG + валидация | 5-10% |
| Fine-tuning alone | НЕ работает |

**Что работает:**
1. RAG — бот только из базы
2. Confidence <70% → эскалация
3. Whitelist для критичных тем
4. Human-in-the-loop

### МИНИМУМ ДАННЫХ:

| Тип | Минимум | Оптимально |
|-----|---------|------------|
| FAQ | 100-500 Q&A | 1000+ |
| Intent (BERT) | **25 на intent = 94%** | 100+ |
| Sales диалоги | 5000+ | 50,000+ |

**Источник:** MIT Media Lab — 25 примеров достаточно для 94% accuracy

**Для Truffles:** 10 интентов × 25 = 250 минимум, 1000 оптимально

### ПРАВИЛЬНЫЕ МЕТРИКИ:

**Deflection Rate — ЛОВУШКА!**
Скрывает фрустрацию клиентов.

**Что мерить:**
- Goal Completion Rate (% достигнутых целей)
- Human Takeover Rate (% эскалаций)
- Abandonment Rate (% брошенных)
- CSAT после бота (1-5)
- Repeat Contact Rate (% повторных)

**Формула Quality Deflection:**
```
Quality = Deflection × CSAT × (1 - Repeat Rate)
```

### ИТОГ:
- 250-1000 примеров собираемо за 2-4 недели
- RAG only, никакой генерации
- Мерить CSAT + Repeat, не deflection
- Один бот = одна задача

## 10. ИЗВЛЕЧЕНИЕ ДАННЫХ ИЗ WHATSAPP

### Почему это важно:
- Реальные фразы клиентов (не выдуманные)
- Сленг и термины специфичные для бизнеса
- Сокращения которые не угадать
- Опечатки которые люди реально делают

### Как получить историю:
1. **WhatsApp Business:** Настройки → Чаты → Экспорт чата
2. **WhatsApp личный:** Открыть чат → ⋮ → Ещё → Экспорт чата
3. **Формат:** .txt файл с датами и сообщениями

### Что искать в истории:
- ТОП-20 вопросов которые повторяются
- Как клиенты называют продукт своими словами
- Специфические термины ниши
- Частые опечатки

### Пример извлечения:
```
Из чата:                    → В базу:
"скок стоит"               → intent: pricing
"прайс скинь"              → intent: pricing  
"че по ценам"              → intent: pricing
"а ваш бот с вацапом?"     → intent: integration
"работает с ватсапом?"     → intent: integration
```

## 11. ТИПИЧНЫЕ ВОПРОСЫ (из исследования)

### До покупки:
1. Сколько стоит? Какие тарифы?
2. Как это работает?
3. Есть пробный период / демо?
4. С чем интегрируется? (WhatsApp, CRM)
5. Сколько времени на внедрение?
6. Есть примеры / кейсы?
7. Чем отличаетесь от конкурентов?
8. Какая поддержка?
9. Что если не подойдёт? Возврат?
10. Нужен ли технический специалист?

### Возражения:
1. "Дорого" → Не видит ценности
2. "Подумаю" → Не срочно
3. "А вдруг не сработает?" → Страх риска
4. "Уже пробовали — не работает" → Негативный опыт
5. "Нет времени" → Занятость
6. "Нужно согласовать" → Не ЛПР
7. "Пришлите на почту" → Вежливый отказ

### После покупки:
1. Как настроить?
2. Как добавить ответы?
3. Почему бот не ответил?
4. Где статистика?
5. Как передать менеджеру?
