## ПРОЦЕСС РАБОТЫ

```
┌─────────────────────────────────────────────────────────────┐
│  1. НАЧАЛО                                                  │
│     Архитектор читает STATE.md → видит план                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. ТЫ ОПИСЫВАЕШЬ ЗАДАЧУ                                   │
│     "Нужно сделать X" или "Проблема Y"                     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. АРХИТЕКТОР ПРОЕКТИРУЕТ                                 │
│     Думает, ссылается на документы, формулирует задачу     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4. КОДЕР РЕАЛИЗУЕТ                                        │
│     Получает задачу, задаёт вопросы, пишет код             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5. КОНЕЦ СЕССИИ                                           │
│     Говоришь "закрыть сессию"                              │
│     Архитектор обновляет STATE.md                          │
└─────────────────────────────────────────────────────────────┘
```

---

## ТВОИ КОМАНДЫ

| Команда | Что происходит |
|---------|----------------|
| **"закрыть сессию"** | Архитектор обновляет STATE.md |
| **"запиши в backlog"** | Идея записывается, не отвлекаемся |
| **"это срочно"** | Добавляется в текущий план |
| **"покажи план"** | Архитектор показывает из STATE.md |
| **"что дальше"** | Архитектор говорит следующую задачу |

## КОГДА ПОЯВИЛАСЬ ИДЕЯ/ХОТЕЛКА

```
Идея появилась
      ↓
  Срочно?
   /    \
 Да      Нет
  ↓       ↓
"Это    "Запиши
срочно"  в backlog"
  ↓       ↓
В план   В STATE.md
         (не отвлекаемся)
```

**Правило:** Не прыгай между задачами. Закончи текущую → потом следующая.

---

## ГДЕ ЧТО ИСКАТЬ

| Вопрос | Где ответ |
|--------|-----------|
| Что сейчас работает/не работает? | `STATE.md` |
| Какой план? | `STATE.md` → ТЕКУЩИЙ ПЛАН |
| Куда записать идею? | `STATE.md` → BACKLOG |
| Как устроен проект? | `STRUCTURE.md` |
| Мои требования? | `STRATEGY/REQUIREMENTS.md` |
| Как должна работать фича X? | `SPECS/*.md` |
| Доступы, команды сервера? | `TECH.md` |

---

## ЧЕГО НЕ ДЕЛАТЬ

1. **Не прыгать между задачами** — закончи одну, потом другая
2. **Не забывать "закрыть сессию"** — иначе контекст потеряется
3. **Не давать задачи кодеру напрямую** — через архитектора
4. **Не игнорировать план** — сначала план, потом хотелки

---

## ЕСЛИ ЧТО-ТО ИДЁТ НЕ ТАК

| Проблема | Решение |
|----------|---------|
| Архитектор несёт хуйню | Скажи прямо, он должен принять |
| Кодер делает не то | Вернись к архитектору, уточни задачу |
| Забыл контекст | Открой STATE.md |
| Не знаю что делать | Скажи архитектору "покажи план" |
| Хочется всё бросить | Скажи "закрыть сессию", отдохни |
